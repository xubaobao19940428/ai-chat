ConversationController API 文档

概述

提供基于角色的会话管理接口，包括创建会话、查询会话列表和详情、获取消息记录，以及分组管理。路径前缀为 /v1，全部接口均需要登录态。

1. 创建新会话

  • 接口描述：基于某个角色创建新的聊天会话
  • HTTP 方法：POST
  • 请求路径：/v1/conversations
  • 是否需要登录：是

1.1 请求参数（Body）

参数名 类型 必填 默认值 说明
character_id int 是 - 角色 ID
title string 否 空字符串 会话标题，不传则可能使用角色名称自动生成
model_id int 否 角色默认模型 会话指定的模型 ID
group_id int 否 0 会话所属分组 ID
meta object 否 {} 会话元数据，结构由业务约定

1.2 响应与错误

  • 角色 ID 非法：invalid_character_id。
  • 角色不存在：character_not_found。
  • 无访问权限：返回 403 类错误，message 为具体原因。
  • 创建失败：create_conversation_failed。

成功时返回新建会话 ID 及角色基础信息：

  • conversation_id：会话 ID。
  • character：角色的 id、name、avatar 等。
  • welcome：角色的欢迎语（如果有）。

2. 获取会话列表

  • 接口描述：按条件分页获取当前用户的会话列表
  • HTTP 方法：GET
  • 请求路径：/v1/conversations
  • 是否需要登录：是

2.1 请求参数（Query）

参数名 类型 必填 默认值 说明
character_id int 否 0 按角色 ID 过滤，不传或 0 表示全部
group_id int 否 0 按分组过滤，0 表示未分组，其他值为分组 ID
page int 否 1 页码
page_size int 否 20 每页数量

2.2 响应说明

成功时 data 为：

  • list：会话列表数组，每项包含会话基本信息（标题、最后一条消息摘要、更新时间等）。
  • page、page_size：分页信息。

3. 获取会话详情

  • 接口描述：获取单个会话的完整信息（不含消息列表）
  • HTTP 方法：GET
  • 请求路径：/v1/conversations/{id}
  • 是否需要登录：是

3.1 请求参数

参数名 位置 类型 必填 说明
id path int 是 会话 ID

3.2 响应与错误

  • 会话 ID 非法：invalid_conversation_id。
  • 会话不存在或不属于当前用户：conversation_not_found_or_forbidden。

成功时 data 为会话详情对象，内部包含会话与角色的关联信息等。

4. 更新会话

  • 接口描述：更新会话的标题、模型、分组等信息
  • HTTP 方法：POST
  • 请求路径：/v1/conversations/update
  • 是否需要登录：是

4.1 请求参数（Body）

参数名 类型 必填 说明
id int 是 会话 ID
title string 否 会话标题
model_id int 否 模型 ID
meta object 否 会话元数据
group_id int 否 分组 ID

4.2 响应与错误

  • 会话 ID 非法：invalid_conversation_id。
  • 更新失败：update_conversation_failed。

成功时 data 为空对象 {}。

5. 删除会话

  • 接口描述：删除指定会话
  • HTTP 方法：POST
  • 请求路径：/v1/conversations/delete
  • 是否需要登录：是

5.1 请求参数（Body）

参数名 类型 必填 说明
id int 是 会话 ID

5.2 响应与错误

  • 会话 ID 非法：invalid_conversation_id。
  • 删除失败：delete_conversation_failed。

成功时 data 为空对象 {}。

6. 获取会话消息列表

  • 接口描述：获取指定会话的历史消息列表
  • HTTP 方法：GET
  • 请求路径：/v1/conversations/{id}/messages
  • 是否需要登录：是

6.1 请求参数

参数名 位置 类型 必填 默认值 说明
id path int 是 - 会话 ID
limit query int 否 0 返回的消息条数，0 表示不限（由服务端控制上限）

6.2 响应与错误

  • 会话 ID 非法：invalid_conversation_id。
  • 会话不存在或不属于当前用户：conversation_not_found_or_forbidden。

成功时 data 为消息数组，从旧到新或由服务端约定的顺序返回。

7. 获取最近会话

  • 接口描述：获取当前用户最近使用的会话列表
  • HTTP 方法：GET
  • 请求路径：/v1/conversations/recent
  • 是否需要登录：是

7.1 请求参数

参数名 位置 类型 必填 默认值 说明
limit query int 否 10 返回的会话数量

7.2 响应说明

成功时 data 为最近使用的会话列表数组。

8. 会话分组管理

8.1 获取分组列表

  • 接口描述：获取当前用户的会话分组列表
  • HTTP 方法：GET
  • 请求路径：/v1/conversations/groups
  • 是否需要登录：是

成功时 data.list 为分组数组，每项包含分组 ID、名称、描述、图标、颜色等字段。

8.2 创建分组

  • 接口描述：创建新的会话分组
  • HTTP 方法：POST
  • 请求路径：/v1/conversations/groups/create

请求参数（Body）：

参数名 类型 必填 默认值 说明
name string 是 - 分组名称，长度限制由服务端约束
description string 否 空字符串 分组描述
icon string 否 空字符串 分组图标（URL 或约定值）
color string 否 空字符串 分组颜色标识
sort_order int 否 0 排序值

成功时返回新分组的 id。

8.3 更新分组

  • 接口描述：更新会话分组信息
  • HTTP 方法：POST
  • 请求路径：/v1/conversations/groups/update

请求参数（Body）：

参数名 类型 必填 说明
id int 是 分组 ID
name string 否 分组名称
description string 否 分组描述
icon string 否 分组图标
color string 否 分组颜色
sort_order int 否 排序值

典型错误：

  • 分组 ID 非法：invalid_group_id。
  • 无更新字段：no_update_data。
  • 更新失败：update_group_failed。

成功时 data 为空对象 {}。

8.4 删除分组

  • 接口描述：删除会话分组
  • HTTP 方法：POST
  • 请求路径：/v1/conversations/groups/delete

请求参数（Body）：

参数名 类型 必填 说明
id int 是 分组 ID

典型错误：

  • 分组 ID 非法：invalid_group_id。
  • 删除失败：delete_group_failed。

成功时 data 为空对象 {}。
