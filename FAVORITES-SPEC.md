# 收藏列表放在哪里？

## 三个方案对比

### 方案 A：侧边栏新增独立入口

在侧边栏加一个 "Favorites" 导航项，跳转到独立的收藏页面。

| 优点 | 缺点 |
|------|------|
| 一级导航，入口可见性最高 | 侧边栏再加一项会显拥挤 |
| 独立页面空间大，可自由排布 | 导航项越多，用户选择成本越高 |

### 方案 B：放在 Library 页面作为 Tab

Library 现有的 "My Assets" 旁边加一个 "Favorites" tab。

| 优点 | 缺点 |
|------|------|
| 不增加导航层级，侧边栏保持简洁 | Library 原本只管素材，职责会变杂 |
| 语义自洽："我保存的东西"和"我标记的东西"同属个人内容 | Tab 切换多一步操作 |
| 统一入口，三种收藏类型在一个页面用筛选切换 | |

### 方案 C：放在各自页面的子 Tab 里

image-generation 的 Creations 旁加 Favorites tab，video-generation 和 explore 同理。

| 优点 | 缺点 |
|------|------|
| 就近访问，在哪收藏就在哪看 | 分散在三个地方，无法统一查看 |
| 不需要新页面 | 每个页面都要加 tab，重复实现 |
| | 想看"所有收藏"没有入口 |

---

## 推荐：方案 B — Library 页面内 Tab

### 理由

1. **用户心智模型**：Library 已经是"我保存的东西"，收藏也是"我标记的东西"，放一起符合预期
2. **不增加导航层级**：侧边栏不用再塞新入口，当前导航结构不变
3. **统一管理**：图片 prompt、视频 prompt、角色三种收藏在一个页面用筛选切分，比分散在三个页面好找
4. **行业参考**：ChatGPT 的 Library 也是把生成内容和收藏放在一起
5. **实现成本低**：只改 Library 一个页面，加 tab 切换 + 收藏列表组件即可
