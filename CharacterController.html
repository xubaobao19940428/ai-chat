CharacterController API 文档

概述

提供 AI 角色的浏览、搜索、创建和管理相关接口。路径前缀为 /v1。

  • 部分接口（角色列表、详情、搜索、热门、最近访问）可在未登录状态使用。
  • 与当前登录用户强相关的接口（我的角色、创建/更新/删除）需要登录。

1. 获取角色列表

  • 接口描述：分页获取角色列表，可按类型、标签、访问类型筛选
  • HTTP 方法：GET
  • 请求路径：/v1/characters
  • 是否需要登录：否

1.1 请求参数

参数名 位置 类型 必填 默认值 说明
page query int 否 1 页码，最小 1
page_size query int 否 20 每页数量，默认 20
type query int 否 - 角色类型过滤，不传表示全部
tag query string 否 空字符串 标签过滤，不传表示全部
access_type query int 否 - 访问类型过滤（公开/私有等），不传表示全部

1.2 响应说明

成功时返回列表和分页信息：

  • list：角色数组，每项为角色概要信息，包含以下字段：
    • id：角色ID
    • name：角色名称
    • avatar：头像URL
    • description：角色描述
    • tags：标签数组
    • chat_count：角色被使用聊天次数
    • creator：创建者信息对象（如果是用户创建）
      • user_id：创建者用户ID
      • username：创建者用户名
      • nickname：创建者昵称
      • avatar：创建者头像
    • 其他字段（character_type、ownership、access_type 等）
  • page / page_size：当前页码与每页数量。

2. 获取角色详情

  • 接口描述：获取指定角色的详细配置和展示信息
  • HTTP 方法：GET
  • 请求路径：/v1/characters/{id}
  • 是否需要登录：否（但部分角色可能需要会员 / 付费权限）

2.1 请求参数

参数名 位置 类型 必填 默认值 说明
id path int 是 - 角色 ID

2.2 响应与错误

  • 当 id 非法时：message 通常为 invalid_character_id。
  • 当角色不存在时：message 通常为 character_not_found。
  • 当当前用户无访问权限（例如仅会员可用）：会返回 403 类错误，message 为具体原因字符串。

成功时 data 为完整角色对象，包括以下字段：

  • 基础信息：id、name、avatar、description、welcome、tags（数组）
  • 配置信息：model_id、model_custom_config、max_history_messages、access_type、is_prompt_public
  • 统计信息：chat_count（角色被使用聊天次数）
  • 创建者信息：creator 对象（包含 user_id、username、nickname、avatar），如果是官方角色则为 null
  • 提示词：prompt（根据权限和配置决定是否包含）
  • 其他字段：created_at、updated_at 等

3. 搜索角色

  • 接口描述：按关键词搜索角色
  • HTTP 方法：GET
  • 请求路径：/v1/characters/search
  • 是否需要登录：否

3.1 请求参数

参数名 位置 类型 必填 默认值 说明
keyword query string 是 空字符串 搜索关键词
page query int 否 1 页码
page_size query int 否 20 每页数量

3.2 响应说明

成功时：

  • list：匹配的角色列表，每项包含 id、name、avatar、description、tags、chat_count 等字段。
  • page / page_size：分页信息。
  • keyword：本次搜索使用的关键词。

当 keyword 为空时会返回参数错误，message 通常为 search_keyword_required。

4. 获取热门角色

  • 接口描述：获取热门角色列表
  • HTTP 方法：GET
  • 请求路径：/v1/characters/popular
  • 是否需要登录：否

4.1 请求参数

参数名 位置 类型 必填 默认值 说明
limit query int 否 10 返回的热门角色数量

4.2 响应说明

成功时 data 为角色数组，按 hot_score（热度值）降序排序，包含以下字段：

  • id、name、avatar、description、tags、welcome
  • chat_count：角色被使用聊天次数

注意：热度值 hot_score 由后台定时任务维护，综合考虑聊天次数、用户反馈等多个维度。

5. 获取最近访问的角色

  • 接口描述：获取当前用户最近访问过的角色
  • HTTP 方法：GET
  • 请求路径：/v1/characters/recent
  • 是否需要登录：是（未登录时通常返回空列表）

5.1 请求参数

参数名 位置 类型 必填 默认值 说明
limit query int 否 10 返回的角色数量

5.2 响应说明

成功时 data 为最近访问角色的数组，按访问时间倒序排列，每项包含：

  • id、name、avatar、description、tags
  • chat_count：角色被使用聊天次数

6. 获取我创建的角色列表

  • 接口描述：获取当前登录用户创建的角色列表
  • HTTP 方法：GET
  • 请求路径：/v1/characters/my
  • 是否需要登录：是

6.1 请求参数

参数名 位置 类型 必填 默认值 说明
page query int 否 1 页码
page_size query int 否 20 每页数量

6.2 响应说明

成功时 data 为：

  • list：当前用户创建的角色列表，每项包含完整的角色信息（包括 chat_count、status、created_at、updated_at 等）。
  • page / page_size：分页信息。

7. 创建角色

  • 接口描述：创建一个新的角色
  • HTTP 方法：POST
  • 请求路径：/v1/characters
  • 是否需要登录：是

7.1 请求参数（Body）

参数名 类型 必填 默认值 说明
name string 是 - 角色名称，长度建议 ≤ 64 字符
description string 否 空字符串 角色描述
avatar string 否 空字符串 头像 URL
prompt string 是 - 系统提示词（角色设定），不能为空
welcome string 否 空字符串 欢迎语
tags array/string 否 空 角色标签，数组或逗号分隔字符串
type int 否 0 角色类型
access_type int 否 2 访问类型，例如私有/公开，默认私有
is_prompt_public int 否 0 是否公开提示词，0/1
model_id int 否 0 默认使用的模型 ID
model_custom_config array 否 [] 模型自定义配置（结构由前后端约定）
max_history_messages int 否 10 最大携带的历史消息条数

7.2 响应与错误

  • 名称为空或过长时：character_name_required / character_name_too_long。
  • 提示词为空时：system_prompt_required。
  • 创建失败时：create_character_failed。

成功时：

{
  "code": 0,
  "message": "success",
  "data": {
    "id": 123
  }
}

8. 更新角色

  • 接口描述：更新已有角色的部分字段
  • HTTP 方法：POST
  • 请求路径：/v1/characters/update
  • 是否需要登录：是

8.1 请求参数（Body）

参数名 类型 必填 说明
id int 是 角色 ID
其他字段 同创建 否 仅会更新传入的字段

8.2 响应与错误

  • id 非法：invalid_character_id。
  • 角色不存在：character_not_found。
  • 当前用户无权管理该角色：no_permission_to_update_character。
  • 更新数据为空：no_fields_to_update。
  • 更新失败：update_character_failed。

成功时返回空数据对象：data 为 {}。

9. 删除角色

  • 接口描述：删除指定角色
  • HTTP 方法：POST
  • 请求路径：/v1/characters/delete
  • 是否需要登录：是

9.1 请求参数（Body）

参数名 类型 必填 说明
id int 是 角色 ID

9.2 响应与错误

  • id 非法：invalid_character_id。
  • 角色不存在：character_not_found。
  • 无删除权限：no_permission_to_delete_character。
  • 删除失败：delete_character_failed。

成功时返回空数据对象：data 为 {}。
