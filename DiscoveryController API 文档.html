DiscoveryController API 文档

概述

DiscoveryController 提供发现/推荐分组及分组内内容的查询接口，支持分页、国家过滤、位置（placement）区分，以及分组预览条目数量限制。

1. 获取推荐分组列表

  • 接口描述：按应用、国家、placement 获取推荐分组列表，每个分组可返回限定数量的 items 作为预览
  • HTTP 方法：GET
  • 请求路径：/v1/discovery
  • 是否需要登录：否

1.1 请求参数

参数名 位置 类型 必填 默认值 说明
placement query string 否 home 展示位置，用于区分不同页面/版位，如 home、discovery
country_code query string 否 US 国家代码（ISO 3166-1）。为空时依次尝试：用户信息、GeoIP，最后兜底 US
page query int 否 1 页码，从 1 开始
page_size query int 否 20 每页数量
item_limit query int 否 10 每个分组预览返回的 item 数量上限，0 表示不限制（会返回全部，慎用）

1.2 业务规则

  • 仅返回 status = 1 的分组。
  • 国家过滤：分组 country_codes 为空或包含该国家代码（模糊匹配）。
  • 分组排序：sort_order 降序，其次 id 降序。
  • 分组分页在数据库层完成（避免一次性加载全部分组）。
  • Items 预览：
    • 若 item_limit > 0，每个分组按 item_limit + 1 拉取，用于判断 has_more；若实际数量大于 item_limit，has_more = true，并只返回前 item_limit 条。
    • Items 排序：sort_order 降序，其次 id 降序。
  • 文案多语言：分组 title 支持 JSON 多语言，按 locale 精确匹配，其次语言前缀匹配，再回退。

1.3 响应数据结构

成功时，data 为对象：

{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 1,
        "title": "Featured Bots",
        "layout": "row",
        "has_more": true,
        "items": [
          {
            "id": 101,
            "group_id": 1,
            "type": "character",
            "related_id": 123,
            "title": "AI 助手",
            "subtitle": "推荐理由",
            "cover": "https://example.com/cover.png",
            "action_url": "app://character/123",
            "params": {},
            "related_data": { /* 关联资源补全字段 */ }
          }
        ]
      }
    ],
    "page": 1,
    "page_size": 20
  }
}

分组字段说明

字段名 类型 说明
id int 分组 ID，对应 app_ai_recommend_groups.id
title string 分组标题（多语言解析结果）
layout string 布局样式：row / grid / list / banner
has_more bool 该分组是否还有更多 items（由 item_limit 判断）
items array 该分组的推荐条目列表（见下表）

Item 字段说明（预览）

字段名 类型 说明
id int 条目 ID，对应 app_ai_recommend_items.id
group_id int 分组 ID
type string 内容类型：character / model / url / 其他扩展类型
related_id int 关联资源 ID（如角色 ID）
title string 展示标题，若为空则回填关联资源名称
subtitle string 副标题/推荐理由，若为空则回填关联资源描述
cover string 封面/图标 URL，若为空则回填关联资源的头像/图标
action_url string 跳转链接，若为空则按类型生成默认跳转（如 app://character/{id}）
params object 透传参数（由 params_json 解析）
related_data object/null 批量加载的关联资源原始数据，用于前端展示或跳转

1.4 错误响应

  • 系统级错误示例：
{
  "code": 10001,
  "message": "system_error",
  "data": {}
}
  • 参数错误示例（如 page 非法）：
{
  "code": 40001,
  "message": "invalid_params",
  "data": {}
}

实际错误码以全局规范为准。

2. 获取指定分组的全部 items（滚动加载）

  • 接口描述：按分组 ID 分页获取该分组的推荐内容，供“更多/全部”场景使用
  • HTTP 方法：GET
  • 请求路径：/v1/discovery/{id}/items
  • 是否需要登录：否

2.1 请求参数

参数名 位置 类型 必填 默认值 说明
id path int 是 无 分组 ID，对应 app_ai_recommend_groups.id
page query int 否 1 页码，从 1 开始
page_size query int 否 20 每页数量

2.2 业务规则

  • 仅当分组存在且 status = 1 时返回数据，否则返回空数组。
  • Items 分页、排序与分组列表中的规则一致（sort_order 降序，其次 id 降序）。

2.3 响应数据结构

成功时，data 为对象：

{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": 201,
        "group_id": 1,
        "type": "character",
        "related_id": 456,
        "title": "旅行家",
        "subtitle": "热门角色",
        "cover": "https://example.com/c2.png",
        "action_url": "app://character/456",
        "params": {},
        "related_data": { /* 关联资源补全字段 */ }
      }
    ],
    "page": 1,
    "page_size": 20
  }
}

字段含义同上表（Item 字段说明）。

2.4 错误响应

  • 分组不存在：返回空列表（list: []）。
  • 其他系统/参数错误：按全局错误码规范返回。

3. 相关数据库表

3.1 app_ai_recommend_groups

字段名 类型 说明
id int(11) UNSIGNED 自增 ID
app_id int(11) UNSIGNED 应用 ID
placement varchar(32) 展示位置（如 home/discovery/自定义）
name varchar(64) 内部管理名称
title varchar(1024) 显示标题（多语言 JSON）
layout varchar(32) 布局：row/grid/list/banner
country_codes varchar(1024) 投放国家（逗号分隔，空为全球）
sort_order int(11) 排序权重（越大越靠前）
status tinyint(1) 状态（0:禁用 1:启用）

3.2 app_ai_recommend_items

字段名 类型 说明
id int(11) UNSIGNED 自增 ID
group_id int(11) UNSIGNED 分组 ID
type varchar(32) 内容类型：character/url/model/image...
related_id int(11) UNSIGNED 关联资源 ID
title varchar(255) 覆盖标题/文案
subtitle varchar(255) 副标题/推荐理由
cover varchar(512) 封面图/图标 URL
action_url varchar(1024) 跳转链接（type=url 时使用）
params_json varchar(2048) 额外参数（JSON）
sort_order int(11) 排序权重（越大越靠前）
status tinyint(1) 状态（0:禁用 1:启用）

响应中仅返回与展示相关的字段，其余字段用于内部投放、过滤与扩展。
