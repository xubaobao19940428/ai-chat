# UX 体验改进清单

> 与 ChatGPT / Claude / Gemini 等主流 AI 产品对比后梳理的体验细节改进项。

---

## 第一批（当前迭代）

### 1. 复制按钮反馈优化
**现状：** 复制消息后只弹底部 toast，用户视线需要离开操作位置。
**目标：** 复制按钮图标临时变为 ✓（1.5s 后恢复），就地反馈，去掉多余的 toast。
**涉及：** `chat/[id].vue` 消息操作栏、`MarkdownContent.vue` 代码块复制按钮

### 3. 键盘快捷键体系
**现状：** 仅支持 `⌘K` 一个快捷键。
**目标：** 参考 ChatGPT / Claude / Gemini 的快捷键设计，只加不与浏览器/系统冲突的组合。需先调研各家方案再确定具体 key binding，避免用户骂娘。
**注意：** 不能乱加，必须逐个验证 macOS / Windows / 主流浏览器下无冲突。
**涉及：** 全局 composable `useKeyboardShortcuts.ts`（新建）

### 4. 对话切换骨架屏
**现状：** 切换对话时只显示一个居中 spinner + "Loading messages"。
**目标：**
- **文本对话：** 仿消息形状的骨架屏（左侧头像圆形 + 右侧不等长文本条），3-5 行交替排列
- **图片生成：** 左侧 prompt 卡片骨架 + 右侧图片占位区域，模拟 GPU 渲染特效（扫描线 + 噪点渐显）
- **视频生成：** 卡片骨架 + 视频区域 GPU 渲染特效

**涉及：** `chat/[id].vue` loading 状态区域

### 6. "正在思考" 状态
**现状：** 等待回复时只显示三个弹跳圆点，无文字提示。
**目标：** 显示 "Thinking..." 文字 + 呼吸灯/脉冲动画。如果后端支持推理过程（thinking tokens），可折叠展开查看推理链。
**涉及：** `chat/[id].vue` 空内容 assistant 消息展示

### 7. 消息操作栏交互优化
**现状：** 用户消息的操作栏悬浮在 `-top-10`，鼠标从消息移到操作栏时容易触发 mouseleave 导致消失。
**目标：** 操作栏紧贴消息气泡边缘（无间隙），或加透明连接区域保持 hover 状态不中断。
**涉及：** `chat/[id].vue` 用户消息 hover 操作栏

### 8. 暗色模式切换过渡
**现状：** 主题切换是瞬间跳变，无动画过渡。
**目标：** 切换时给全局加短暂的 `transition`，背景色和文字色平滑过渡（~0.3s），过渡结束后移除 transition 避免影响其他交互性能。
**涉及：** `assets/css/main.css`、`stores/ui.ts`

### 9. 编辑消息未保存提示
**现状：** 编辑模式下点击外部区域直接取消，编辑内容丢失无提示。
**目标：** 内容有变更时，点击外部弹出确认弹窗："Discard changes?"，确认丢弃 / 取消继续编辑。
**涉及：** `chat/[id].vue` `startEditing` / `cancelEditing`

### 10. Toast 自动消失
**现状：** Toast 需要手动点击关闭。
**目标：** 默认 3 秒后自动消失（error 类型 5 秒），底部显示倒计时进度条，hover 时暂停倒计时。
**涉及：** `components/Toast.vue`、`stores/ui.ts`

## 后续规划

### 11. 新对话空状态引导（大改）
**现状：** 新对话页面有 suggestion 按钮，但整体引导不足。
**规划：** 这个后面要大改，不能把用户当傻瓜，要给足提示——新手引导流程、功能亮点展示、场景化入口等。当前阶段暂不动，等整体方案确定后统一改。
**状态：** 📋 规划中

### 12. 消息时间分组
**现状：** 消息列表没有时间分隔线。
**评估：** 聊天主区域加时间分组意义不大，用户关注的是内容不是时间。如果后续做对话内搜索，可以在搜索结果里按时间分组展示。
**状态：** ⏸️ 暂不做

## 第二批（2026-03-25 代码审查）

### 21. 生成失败重试机制
**现状：** 图片/视频生成失败后只显示错误状态，用户无法一键重试。
**目标：** 失败卡片上加"重试"按钮，点击后用相同参数重新发起生成。
**涉及：** `image-generation.vue`、`video-generation.vue`、`chat/[id].vue`

### 22. 模型列表加载失败兜底
**现状：** `ModelSelector.vue` 网络错误时 spinner 无限转，无错误态。
**目标：** 加载失败时显示错误提示 + 重试按钮，替代无限 loading。
**涉及：** `ModelSelector.vue`、`stores/models.ts`

### 23. 移动端 ModelSelector 宽度适配
**现状：** 下拉固定 320px 宽度，小屏设备可能超出视口。
**目标：** 宽度改为 `min(320px, 90vw)`。
**涉及：** `ModelSelector.vue`

### 24. 移动端键盘遮挡输入框
**现状：** `UnifiedInput.vue` 固定底部定位，手机键盘弹出时可能被覆盖。
**目标：** 使用 `visualViewport` API 或 CSS `env(keyboard-inset-height)` 适配键盘高度。
**涉及：** `UnifiedInput.vue`

### 25. 瀑布流图片加载布局跳动
**现状：** 图片加载前无固定宽高比占位，加载后布局跳动。
**目标：** 用 `aspect-ratio` CSS 或固定尺寸容器在骨架阶段预留空间。
**涉及：** `image-generation.vue`、`video-generation.vue`、`MasonryGrid.vue`

### 26. ModelSelector 键盘导航
**现状：** 下拉列表只能鼠标操作，不支持键盘。
**目标：** 支持 ↑↓ 键移动焦点、Enter 选择、Esc 关闭。
**涉及：** `ModelSelector.vue`

### 27. Icon-only 按钮无障碍标签
**现状：** 语音输入、附件、发送等纯图标按钮缺少 `aria-label`。
**目标：** 所有 icon-only 按钮补齐 `aria-label`。
**涉及：** `UnifiedInput.vue`、`chat/[id].vue`、`Sidebar.vue`

### 28. ConfirmDialog 焦点管理
**现状：** 弹窗打开后不会自动 focus 到主操作按钮，需要用户手动 Tab。
**目标：** 弹窗打开时自动 focus 到确认按钮，Esc 关闭。
**涉及：** `ConfirmDialog.vue`

### 29. 侧边栏空状态引导
**现状：** 无会话时只显示一行文字。
**目标：** 加引导插图 + 操作提示（"开始新对话"按钮或功能入口卡片）。
**涉及：** `Sidebar.vue`

### 30. 素材库空状态引导
**现状：** 无资源时缺少上传引导。
**目标：** 居中空态插图 + 支持格式说明 + 上传按钮。
**涉及：** `library.vue`

### 31. Explore 搜索空状态
**现状：** "No bots found" 过于简陋。
**目标：** 加图标、大标题、建议操作（"换个关键词试试"、"浏览全部分类"）。
**涉及：** `explore.vue`

### 32. 文件上传进度条
**现状：** 上传只有 boolean 状态，无进度反馈。
**目标：** 加进度百分比或进度环动画。
**涉及：** `UnifiedInput.vue`、`composables/useFileUpload.ts`

### 33. 拖拽区域格式提示
**现状：** 拖拽覆层只显示"拖放文件到此处"，不标注支持的文件类型。
**目标：** 根据当前模型能力显示支持的格式（如"支持 JPG、PNG、WebP"）。
**涉及：** `chat/[id].vue`

### 34. 新消息到达提示
**现状：** 用户向上滚动时，新消息到达没有提示。
**目标：** 显示 "↓ N 条新消息" 浮动按钮，点击滚动到底部。
**涉及：** `chat/[id].vue`

### 35. 无限滚动到底标识
**现状：** `explore.vue` "No more data" 文字太小。
**目标：** 用分隔线 + 居中图标 + "已全部加载" 文案，视觉更明确。
**涉及：** `explore.vue`、`image-generation.vue`、`video-generation.vue`

### 36. 侧边栏折叠过渡动画
**现状：** 宽度有 transition 但内容切换生硬。
**目标：** 内容在折叠/展开过程中加 fade 淡出淡入。
**涉及：** `Sidebar.vue`

### 37. 标签页切换过渡
**现状：** Inspiration / Creations 标签页切换瞬间跳变。
**目标：** 加 fade 或 slide 过渡动画。
**涉及：** `image-generation.vue`、`video-generation.vue`

### 38. 消息出现动画优化
**现状：** TransitionGroup 动画偏快。
**目标：** 延长到 400ms，加 fade + 轻微 translateY 效果。
**涉及：** `chat/[id].vue`

### 39. 已编辑消息标记
**现状：** 编辑过的消息外观与原消息无区别。
**目标：** 消息时间旁显示 "(已编辑)" 标签。
**涉及：** `chat/[id].vue`

### 40. 长代码块折叠
**现状：** 所有代码块完整展开，长代码块占据大量视口。
**目标：** 超过 15 行的代码块默认折叠（显示前 10 行），可点击展开。
**涉及：** `MarkdownContent.vue`、`markdown.css`

### 41. Explore 搜索 debounce
**现状：** 每次按键立即触发过滤。
**目标：** 加 300ms debounce 减少无效重渲染。
**涉及：** `explore.vue`

### 42. Explore 筛选状态持久化
**现状：** 选中的分类标签刷新后丢失。
**目标：** 持久化到 localStorage 或 URL query。
**涉及：** `explore.vue`

### 43. 侧边栏虚拟滚动
**现状：** 会话列表全量渲染，100+ 时可能卡顿。
**目标：** 引入虚拟滚动（如 `vue-virtual-scroller`），只渲染可见区域。
**涉及：** `Sidebar.vue`

### 44. 参数字段说明 tooltip
**现状：** aspect ratio、guidance scale 等专业参数无说明。
**目标：** 参数标签旁加 ⓘ 图标，hover 显示简短说明。
**涉及：** `ImageGenerationParams.vue`、`VideoGenerationParams.vue`

### 45. disabled 按钮样式统一
**现状：** 不同组件的 disabled 样式不一致（opacity-20 / opacity-50 / 无变化）。
**目标：** 统一为 `opacity-50 cursor-not-allowed` 标准样式。
**涉及：** `UnifiedInput.vue`、`ModelSelector.vue` 及全局
