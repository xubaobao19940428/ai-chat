ModelController API 文档

本文档详细说明 ModelController 提供的 API 接口，按照现有功能包括：

  1. 获取模型支持的参数配置
  2. 获取模型列表及其能力信息
  3. 获取所有可用的 Provider 列表
  4. 获取所有可用的能力列表

公共说明

  • 统一返回格式使用 support\Response 封装：
    • 成功：Response::success(data)
    • 失败：Response::error(message, code)
  • 错误码参考项目通用规范

1. 获取模型支持的参数配置

  • 接口：GET /models/supported-params
  • 说明：根据模型、能力、供应商和 API 调用方式，获取该组合支持的参数。

请求参数

参数 类型 必填 说明
provider string 是 模型供应商，例如 openai
model string 是 模型标识，例如 gpt-4o
capability string 是 模型能力，例如 image_generation
api_format string 否 API 调用方式，默认 native

响应示例

{
  "code": 0,
  "message": "success",
  "data": {
    "provider": "openai",
    "model": "gpt-4o",
    "capability": "chat",
    "api_format": "native",
    "params": {
      "temperature": {"type": "slider", "default": 0.7},
      "max_tokens": {"type": "number", "default": 2048}
    }
  }
}

错误码

错误码 信息 说明
40001 missing_required_params 缺少必要参数

2. 获取模型列表

  • 接口：GET /models
  • 说明：按供应商和能力筛选模型，并返回其详细能力信息。

请求参数

参数 类型 必填 说明
provider string 否 按供应商过滤，如 openai
capability string 否 按能力过滤，如 chat

响应示例

{
  "code": 0,
  "message": "success",
  "data": [
    {
      "provider": "openai",
      "model": "gpt-4o",
      "display_name": "GPT-4o",
      "description": "旗舰多模态模型",
      "capabilities": ["chat", "vision", "tools"],
      "context_length": 128000,
      "max_output_tokens": 16384,
      "is_default": true,
      "execution_mode": "sync"
    }
  ]
}

3. 获取 Provider 列表

  • 接口：GET /models/providers
  • 说明：汇总所有模型对应的供应商列表及各自模型数量。

响应示例

{
  "code": 0,
  "message": "success",
  "data": [
    {
      "name": "openai",
      "display_name": "Openai",
      "model_count": 12
    }
  ]
}

4. 获取能力列表

  • 接口：GET /models/capabilities
  • 说明：列出所有模型所支持的能力，并统计每种能力对应的模型数量。

响应示例

{
  "code": 0,
  "message": "success",
  "data": [
    {
      "name": "chat",
      "display_name": "Chat",
      "model_count": 18
    }
  ]
}

备注

  • 所有接口默认支持 GET 请求
  • 根据需要可在请求头中附加业务必要的认证信息
  • 返回的信息中的字段量以当前 ModelController 中实际返回为准
